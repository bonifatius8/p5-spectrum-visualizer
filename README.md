# p5-spectrum-visualizer

マイク入力をリアルタイム FFT 解析し、周波数スペクトラムをグラフィックで可視化するウェブアプリ。

**デモ:** [https://bonifatius8.github.io/p5-spectrum-visualizer/](https://bonifatius8.github.io/p5-spectrum-visualizer/)

## 使い方

1. 画面をクリック → 初回はブラウザがマイク入力の許可を求める
2. 許可後、マイクからの音声を FFT 解析し描画開始
3. 周波数成分・音量に応じてグラフィックがリアルタイム変化

---

## プロジェクト概要

このプロジェクトは、Vite、TypeScript、p5.js、Tone.js を使用して、マイク入力などの音声データをリアルタイムに可視化するウェブアプリケーションです。主要なコンポーネントは、p5.js による描画部分と、Tone.js を使用した音声入力および FFT 解析部分です。

## 開発環境セットアップ

1.  リポジトリをクローンします。
2.  プロジェクトルートディレクトリで開発サーバーを起動します。`npm run dev`

`http://localhost:<ポート番号>/` に Chrome などのモダンブラウザでアクセスしてください。マイクへのアクセス許可を求められたら許可してください。音声入力に応じて画面が変化することを確認できます。

## プロジェクト状況

このプロジェクトは、Vite、TypeScript、p5.js、Tone.js を使用して、マイク入力などの音声データをリアルタイムに可視化するウェブアプリケーションです。

主な機能：

-   マイク入力からの音声データの取得と FFT 解析
-   周波数データに基づいた ellipse（ボール）の描画
-   スペクトラムアナライザの描画
-   ウィンドウリサイズ時のキャンバスサイズの追従
-   物理シミュレーションによる ellipse（ボール）の挙動制御
-   残像効果の追加
-   「音声処理を開始」ボタンの実装（マイクアクセス許可問題の対策）
-   `sketch`の`draw`関数で行われている FFT データを使い、ピーク検出を行う。今後 FFT データを使って様々な機能を作ることを考慮し、ピーク検出機能を API として利用できるように、`draw`関数内で走るマイクロサービスのように実装する。

## SOLID 原則

-   **SRP (単一責任の原則):** `ellipse.ts`から物理シミュレーションロジックを`ellipsePhysics.ts`に分離、`sketch.ts`から FFT 解析ロジックを`spectrumAnalyzerClass.ts`に分離しました。

## 今回の方針

-   `sketch`をシンプルに保つために、`draw`で行う毎フレームの FFT を渡してデータ処理の管理をするモジュールが必要。
-   `peakAnalyzer`もそのデータ処理管理モジュールから処理を呼ぶ。
-   `draw`に多量の変数を伴い呼び出しを行うような文を置かないようにする。

## 今後の方針

-   これからの機能開発では、クラスではなく関数モジュールを使用する。

## 今回の方針

-   描画モジュールから参照できる処理済みデータのプールのような機能を作り、データ処理管理で処理したデータを保持する。
-   `draw`関数をシンプルに保つために、データ管理モジュールで FFT を行い、データ処理モジュールで処理した結果をデータプールに格納し、描画モジュールはそのデータプールからデータを取得するようにする。
-   `draw`の毎フレームの流れは、
    -   データ処理モジュール →（データ処理モジュールからデータプールにデータ格納）→
    -   描画モジュールをよびだしフレームバッファに描画 →
    -   フレームバッファを plane にテクスチャ貼り付けしてメインカンバスに描画
